Назва: Варіант використання «Сформувати звіт про тварин» (частина модуля «Адміністратор»).

Мета: Надання користувачу з роллю «Адміністратор» інструменту для отримання зведеної інформації про стан тварин у зоопарку. Модуль імітує запит до бази даних та генерує текстовий звіт на основі критерію, введеного користувачем (наприклад, за типом тварин).

Опис алгоритму роботи

Алгоритм роботи програми є лінійним і складається з трьох логічних етапів:

Ввід:

Програма запускається.

Викликається функція getUserInput(), яка виводить на екран запит Enter animal type....

Система очікує, доки користувач введе рядок тексту та натисне Enter.

Функція зчитує весь рядок (за допомогою getline) і повертає його у головну функцію main().

Обробка:

Функція main() отримує рядок від getUserInput() і передає його у функцію generateReport().

generateReport() аналізує отриманий рядок за допомогою блоку if...else if...else.

Якщо рядок відповідає відомому критерію (напр., "Mammals"), функція "генерує" (створює) новий рядок string, що містить заздалегідь підготовлені дані звіту.

Якщо введений рядок не відповідає жодному критерію, функція генерує рядок з повідомленням про помилку.

Згенерований рядок повертається у main().

Вивід:

Функція main() отримує рядок звіту від generateReport() і передає його у функцію printReport().

printReport() форматує вивід (додає заголовки, рамки) та друкує фінальний звіт на консоль.

Фрагменти коду
Фрагмент 1: Головна функція (main), що демонструє логіку "Ввід-Обробка-Вивід"
//Основна програма
int main() {
    // 1. Ввід
    string reportType = getUserInput();

    // 2. Обробка
    string reportData = generateReport(reportType);

    // 3. Вивід
    printReport(reportData);

    return 0;
}

Фрагмент 2: Функція обробки (generateReport), що демонструє логіку вибору
// Функція ОБРОБКИ 
// Функція генерує звіт на основі запиту
string generateReport(string reportType) {
    string generatedData;

    // Імітація запиту до БД на основі введеного типу
    if (reportType == "Mammals" || reportType == "mammals") {
        generatedData = "Type: Mammals\n";
        generatedData += " - Lions: 5 (Healthy)\n";
        generatedData += " - Tigers: 3 (1 under review)\n";
    } else if (reportType == "Birds" || reportType == "birds") {
        generatedData = "Type: Birds\n";
        generatedData += " - Macaw Parrots: 12 (Healthy)\n";
    } else {
        // Обробка невідомого запиту
        generatedData = "Error: Report for type '" + reportType + "' not found.";
    }
    
    return generatedData;
}

Пояснення логіки реалізації
Реалізація відповідає вимозі завдання щодо модульності. Замість того, щоб писати весь код в main(), логіка була декомпозована на три окремі функції:

getUserInput(): Ця функція приховує всю логіку, пов'язану з отриманням даних. Вона використовує getline(cin, type), а не cin >> type, щоб коректно обробляти ввід, який може містити пробіли (наприклад, "African Mammals").

generateReport(): Це "мозок" модуля. Він отримує дані та повертає результат. Це чиста функція, яка не взаємодіє з консоллю. Це робить її легкою для тестування: можна перевірити її роботу, не запускаючи всю програму. Використання if...else if імітує логіку switch або пошук у базі даних.

printReport(): Ця функція відповідає лише за гарне форматування виводу. Якщо завтра знадобиться змінити вигляд звіту (наприклад, додати більше ====), нам потрібно буде змінити лише цю функцію, не торкаючись main або generateReport.

Цей підхід (розділення відповідальності) робить код чистим, легким для читання та майбутніх модифікацій.

Короткий опис взаємодії модуля з іншими частинами системи

У контексті повної системи «Управління Зоопарком» цей модуль:

Актор: Ініціюється Адміністратором.

Взаємодія: У реальній системі функція generateReport() не мала б заздалегідь прописаних даних. Натомість вона б зверталася до Модуля Бази Даних (БД).

Потік даних:

generateReport() відправляє SELECT запит до Модуля БД (наприклад, SELECT * FROM animals WHERE type = 'Mammals').

Модуль БД повертає список тварин.

generateReport() обробляє цей список і форматує його у string.

Вихід: Згенерований звіт надається Адміністратору, який на його основі може приймати рішення та використовувати інші модулі (наприклад, Надавати доступ... або Керувати вольєрами).
